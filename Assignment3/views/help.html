Endpoints

GET Requests

Retrieve Users
Method: GET URL: /Users/
Description: Retrieves all of the users currently stored in the system and sends them in a response as a JSON array.
Input: none
Output: JSON array of all users currently in the system
Retrieve User by ID
Method: GET URL: /Users/{userId}
Description: Uses the userId field to retrieve a User record from the database.
Input: userId passed in the URL as a parameter.
Output: Response sends the user the JSON User object if the userId corresponds to a User in the database. If the userId cannot be found, a 404 error is returned.

Retrieve Posts by User
Method: GET URL: /Users/Posts/{userId}
Description: Retrieves all of the posts currently stored in the system that the specified User has authored and sends them in a response as a JSON array.
Input: Request with a valid authentication token in the header that corresponds to the user specified by the userId in the parameters of the URL.
Output: JSON array of all posts currently in the system authored by the authenticated user sent in a response with a 200 status code. If the request does not contain a valid authentication token for the specified User, a response with status code 401 will be sent to the client. If no Posts authored by the specififed User can be found, a response with status code 404 will be sent.
Retrieve Posts
Method: GET URL: /Posts/
Description: Retrieves all of the posts currently stored in the system and sends them in a response as a JSON array.
Input: none
Output: JSON array of all posts currently in the system
Retrieve Post by ID
Method: GET URL: /Posts/{PostId}
Description: Uses the postId field to retrieve a Post record from the database.
Input: postId passed in the URL as a parameter.
Output: Response sends the client the JSON Post object corresponding to the postId specified in the url if it could be retrieved from the database with a status code of 200. If the Post with postId cannot be found, a 404 error is returned.

POST Requests

Create new User

URL: /Users

Description: Used to create a new User record (unless a user with that userId is already in the database).

Input: userId passed in the URL as a parameter.

Output: If the userId submitted in the body of the request does not match an existing user in the database and the user was able to be succesfully created, the new User object is returned as a JSON object with a status of 200. If the userId already corresponds to an existing User in the database, an error message with status code 409 will be sent. If for any other reason a User was unable to be successfully created and added to the database, a response with status code 404 will be sent.

Create new Post

URL: /Posts

Description: Used to create a new Post record.

Input: postId passed in the URL as a parameter, as well as the post title, content, userId of the user authoring the post, and the headerImage (a string representing the url that will link to the image that will be displayed in the header of the blog post page).

Output: If the header for the request contains a valid authentication token and the Post was able to be created succesfully then a response with status code 201 will be sent to the client. If the request header does not contain a properly formatted authenication token then no post will be created and the client will receive a response with status code 401.

PATCH Requests

Update existing User information

URL: /Users/{userId}

Description: Used to update an existing User record's fields (except for the userId field).

Input: the request provides the userId in the URL as a parameter for the user that should be updated. If a use matching that userId exists in the database, all of the User attributes passed in the body of the request will be updated, the rest will remain as is.

Output: The response is a JSON object for the User object with the updated fields if the user was able to be updated. If not, a 404 error is received instead.

Update existing Post information

URL: /Posts/{postId}

Description: Used to update an existing Post's content or/and headerImage attributes.

Input: The request should contain a valid authentication token for the author of the Post that is being patched. The request's postId parameter is used to query the database to find the post to be patched once the authentication token has been validated. The request's body should contain the content and headerImage updates that the user wishes to patch the post with (either attribute can be patched seperately or both can be patched at once).

Output: The response is a JSON object for the Post that was patched if the Posts' author was properly auhtenticated able the post was updated. In this case the client will receive a response with a 200 status code. If the user was not properly authenticated the response will be sent with a status code of 401. If the post with postId equal to the one passed in the request's URL as a paremeter is unable to be found in the database, a status code of 404 will be sent to the client instead.

DELETE Requests

Delete existing User

URL: /Users/{userId}

Description: Used to delete an existing User record.

Input: The userId for a user in the database that should be deleted. This value is passed as a parameter in the URL.

Output: If the user was succesfully deleted, a response with a positive status message will be sent. Otherwise, a 404 error will be sent.

Delete existing Post

URL: /Posts/{postId}

Description: Used to delete an existing User record.

Input: The request should contain the postId that the user wishes to delete as a parameter in the url and a valid authentication token in the header of the request.

Output: If the user was succesfully authenticated using the authentication token and is the author of the post that they wish to delete, the post will be deleted from the database and the client will receive a response with status code 204. If the authorization token was invalid and the user was not able to be authenticated, the client will receive a response with status code 401. If the post in question was unable to be found, the client will receive a response with status code 404.
